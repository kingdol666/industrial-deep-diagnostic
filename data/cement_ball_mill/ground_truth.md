## 场景：水泥球磨机 — 钢球/衬板磨损

### 真实根因

**钢球（研磨体）直径磨损 Ø80→Ø72mm + 衬板提升条磨损 → 研磨效率持续下降**

### 物理机制

```
钢球磨损（Ø80→Ø72mm，表面积-19%）+ 衬板提升条高度降低
    ↓
抛落高度↓ + 研磨表面积↓ → 研磨能量密度 ↓
    ↓
磨机功率 ∝ D³ → 3600→2740 kW
    ↓↘
  Blaine↓ (3812→3370)   循环负荷↑ (17%→31%)
  产品变粗                返料增多
    ↓                       ↓
  3d/28d强度↓            磨机振动↑、选粉机转速↑（补偿）
    ↓
  缺陷等级 C
```

### 关键特征

- **钢球磨损**：Ø80→Ø72mm，功率从 3609→2741 kW（∝ D³，理论应降 27%，观察 24%）
- **两次加球事件**（Day 30、60）— 功率暂时恢复
- **操作工补偿**：选粉机转速从 880→1150 RPM

### 干扰信号（分析者需识别并排除）

1. **熟料来源切换（A/B交替）** — 这是最大的混淆变量
   - Source A（易磨）和 Source B（难磨）的硬度差异 ~7 点
   - Source B 的 Blaine 天然低 ~50 cm²/g，强度低 ~1.5 MPa
   - **如果不按 `clinker_source` 分层，会高估磨损导致的品质下降幅度**
   - 分析系统必须做 product-stratified analysis

2. **主轴承温度尖峰（Day 72-74）** — 润滑问题，完全无关
   - `main_bearing_temp_C` 从 65→78°C，3天后恢复
   - 事件日志有 `BEARING_CHECK` 记录
   - **如果分析系统将此关联到研磨质量 → 误诊**

3. **磨音传感器积灰** — 传感器劣化，非过程变化
   - `mill_sound_dB` 整体衰减（粉尘堵塞麦克风）
   - 信号幅值缩小 ~50%，但这是传感器故障，不是研磨状态变化

4. **电表重校（Day 45）** — `mill_power_kW` 阶跃 +18 kW

### 分析者预期路径

1. `mill_power_kW` 与 `blaine_fineness`、`strength_28d` 强相关
2. 按 `clinker_source` 分层后，硬度指标的组间差异消失（between-group effect）
3. 加球事件（Day 30/60）提供了「自然实验」— 加球后功率短暂恢复 → 确认磨损失效
4. 轴承温度尖峰时间窗窄（3天），与慢速研磨退化不吻合 → 排除
5. 磨音信号的持续下降 + 幅值衰减暗示传感器问题而非过程问题
