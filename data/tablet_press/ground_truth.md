# 真实根因说明（禁止泄露给诊断分析系统）

## 场景：制药旋转压片机 — 3号站冲头尖端磨损

### 数据文件说明

| 文件 | 内容 | 结构 |
|------|------|------|
| `batch_record_2025Q4.csv` | 批记录（80批，每批1行） | 过程参数、环境参数 |
| `tablet_test_results_2025Q4.csv` | 片剂检测结果（~1,900行，~60%覆盖率） | 每片检测值 |
| `production_event_log_2025Q4.csv` | 生产事件日志 | 事件驱动 |

### 真实根因

**3号压片站上冲头尖端渐进磨损（R10.0→R8.2mm）→ 压缩效率降低 ~25%**

### 物理机制

```
3号站冲头磨损（不可观测，需按 station 分层来看）
冲头球面 R10.0→R8.2mm（尖部变平）
    ↓
接触面积↑ → 同压缩力下比压↓
    ↓
药片密度不均匀（中心疏松）+ 颗粒结合力不足
    ↓
硬度↓（100→~75N） + 脆碎度↑（0.3%→>1%）+ 厚度↑
    ↓
缺陷等级 B/C（仅发生在 3 号站）
    ↓
1 号站和 2 号站：始终正常（冲头未磨损）
```

### 关键特征

- **根因是站号相关的** — 如果不按 `station` 分层，整体数据几乎看不出退化
- **Batch ~40 的冲头修磨**（`PUNCH_DRESSING`）— 部分恢复，但未完全修复
- **操作工补偿**：压缩力从 18→28 kN（补偿行为，反向因果！）

### 干扰信号（分析者需识别并排除）

1. **混合水分波动（全时段随机）** — 影响所有站的硬度
   - 水分高 → 暂时硬度高（掩盖 3 号站的恶化）
   - 水分低 → 硬度低（可能误判为冲头问题）
   - **全随机，无时间趋势** — 但如果不控制会引入噪声

2. **硬度仪零漂（~-0.03N/批，全时段）** — 传感器漂移
   - 末期硬度比实际偏低 ~2.4N
   - **非常关键的陷阱**：如果分析者没注意到硬度仪漂移，可能把仪器漂移归因为过程退化

3. **压缩区温度传感器响应滞后** — 温度变化落后于实际压缩事件
   - 构建时间序列时，温度的变化似乎"晚于"质量变化
   - 容易误导分析者认为"温度不是原因"
   - 实际上是传感器时间常数造成的伪影

4. **润滑剂批次变更（Batch ~50）** — `LUBRICANT_CHANGE`
   - 可能短暂影响片剂质量，但不是根因

### 分析者预期路径

1. **第一步**：整体数据各指标与过程参数弱相关 → 提示存在未被观测的分组变量
2. **关键发现**：按 `station` 列分组后，站3的 `hardness_N` 随时间显著下降，站1和站2稳定
3. **排除混淆**：
   - 水分随机波动（站1/2/3同步，站1/2稳定→水分效应不影响根因判断）
   - 硬度仪漂移（同时影响所有站，但站1/2保持不变→漂移效应很小）
   - 压缩力上升（站1/2硬度不变→力的增加不是硬度下降的原因）
4. **物理验证**：R10→R8.2 的有效面积变化 → 比压降低约 25% → 硬度下降幅度吻合
5. **结论**：3号站冲头磨损 → 需更换该站冲头

### 设计难度说明

这是三个场景中诊断难度最高的，因为它要求分析系统：
1. 识别出 `station` 是关键的分层变量
2. 不做分层时看到弱相关不轻易下结论
3. 识别操作工补偿（压缩力上升是结果不是原因）
4. 排除随机水分和仪器漂移的干扰
